# Cashback Support Process

Пет-проект по BPM-аналитике: анализ и оптимизация процесса обработки обращений клиентов по вопросам начисления кэшбэка.

## Описание проекта

В проекте смоделирован процесс, в котором клиент обращается в поддержку, если не понимает статус начисления кэшбэка.

Основная проблема: клиент не всегда видит в приложении, начислен ли кэшбэк, ожидается ли он или задерживается. Из-за этого часть клиентов повторно обращается в поддержку.

## Цель проекта

Выявить причины повторных обращений клиентов и предложить улучшение бизнес-процесса, которое снизит нагрузку на операторов и улучшит клиентский опыт.

## Что сделано

- Создана учебная база данных в PostgreSQL.
- Добавлены синтетические данные по клиентам, операциям кэшбэка и обращениям в поддержку.
- Проведён SQL-анализ обращений.
- Выявлены сценарии, в которых чаще возникают повторные обращения.
- Подготовлена BPMN-схема текущего и целевого процесса.

## Инструменты

## Основные выводы

По результатам анализа смоделированного датасета повторные обращения возникали в ситуациях, когда кэшбэк находился в ожидании или был начислен с задержкой, а клиент не видел понятный статус операции в приложении.

Простые обращения может закрывать бот, потому что такие вопросы не всегда требуют участия оператора. Операторам лучше передавать только нестандартные случаи, где нужна дополнительная проверка.

## Предложенное улучшение

Добавить в приложение понятный статус начисления кэшбэка и ожидаемую дату начисления.  
Если вопрос типовой, бот должен автоматически объяснять клиенту статус операции.  
Если ситуация нестандартная, обращение передаётся оператору вместе с данными по операции.
